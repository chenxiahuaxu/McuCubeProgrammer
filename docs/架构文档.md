# MCU Cube Programmer — 架构文档

> 版本: v0.0.1 | 日期: 2026-07-03

---

## 1. 系统概览

MCU Cube Programmer 是一个跨平台 MCU 烧录图形化工具，基于 pyOCD + Flet 构建，支持 ST-Link / DAPLink / J-Link 等多种调试探针和 STM32 / GD32 / NXP / Nordic 等 Arm Cortex-M 芯片。

```
┌────────────────────────────────────────────────────────────┐
│                      用户界面层 (Flet)                      │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────┐ │
│  │FlashTab │ │ProbeTab │ │ SwoTab  │ │ LogTab  │ │SetTab│ │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘ └──────┘ │
│       │           │           │           │                │
│  ┌────┴───────────┴───────────┴───────────┴────────────────┤
│  │                   组件层 (components/)                   │
│  │  ProbeSelector │ TargetSelector │ FlashPanel │ LogView │
│  └──────────────────────┬──────────────────────────────────┘ │
├────────────────────────────────────────────────────────────┤
│                      逻辑层 (logic/)                        │
│  ┌──────────────┐ ┌──────────────┐ ┌────────────────────┐  │
│  │ ProbeManager │ │TargetManager │ │  FlashController   │  │
│  └──────┬───────┘ └──────┬───────┘ └────────┬───────────┘  │
├─────────┼────────────────┼──────────────────┼──────────────┤
│         │           后端层 (backend/)        │              │
│         │  ┌──────────────────────────────┐ │              │
│         └──│         PyOCDBackend         │──┘              │
│            │     (实现 BackendABC)         │                 │
│            └──────────────┬───────────────┘                 │
├───────────────────────────┼─────────────────────────────────┤
│                           │                                 │
│                    pyOCD SDK                                 │
│      ConnectHelper / FileProgrammer / SWVReader / ...       │
└────────────────────────────────────────────────────────────┘
```

## 2. 分层设计

### 2.1 UI 组件 (`src/ui/components/`)

纯展示组件，**不导入逻辑层**，通过回调与外界通信。

| 组件 | 职责 | 回调注入 |
|------|------|---------|
| `ProbeSelector` | 探针下拉框 + 刷新按钮 | `on_refresh` / `on_probe_selected` |
| `TargetSelector` | 厂家 + 芯片两级联动 + Pack 安装 | `on_target_selected` / `on_pick_pack` |
| `FirmwareFilePicker` | 固件文件选择 (bin/hex/elf/axf) | `on_file_selected` |
| `FlashPanel` | 擦除/烧录/取消按钮 + 进度条 + 状态 | `on_erase` / `on_flash` / `on_cancel` |
| `LogView` | 等宽字体日志视图 + 彩色级别标记 | 无回调（被动数据写入） |

### 2.2 Tab 页面 (`src/ui/tabs/`)

编排层，组装组件并连接逻辑层。**FlashTab 是唯一允许导入逻辑层的 UI 组件。**

| Tab | 职责 |
|-----|------|
| `FlashTab` | 烧录工作流编排，连接所有子组件到逻辑层 |
| `ProbeTab` | 探针详细信息卡片列表 |
| `SwoTab` | SWO 串行调试输出，独立 LogView |
| `LogTab` | 全屏日志视图，清除/导出功能 |
| `SettingsTab` | 主题切换 + 关于信息 |

### 2.3 App 主类 (`src/app.py`)

应用生命周期管理，**唯一持有所有层实例的对象**：

```
App
├── backend: PyOCDBackend           (后端)
├── probe_manager: ProbeManager     (逻辑)
├── target_manager: TargetManager   (逻辑)
├── flash_controller: FlashController (逻辑)
├── log_view: LogView               (共享 UI)
├── flash_tab / probe_tab / swo_tab / log_tab / settings_tab
└── loop: asyncio.AbstractEventLoop
```

启动流程：
```
main.py → App.__init__(page)
  → _detect_platform()      # 检测 windows/macos/linux/web
  → _configure_window()     # 标题 + 尺寸
  → _configure_theme()      # 深色 + PCB 配色
  → _init_backend()         # 创建 PyOCDBackend + 三个 Manager
  → _init_log_view()        # 创建 LogView + 注册 Logger 回调
  → _build_tabs()           # 创建 5 个 Tab 页面
  → _auto_scan_probes()     # 后台任务：扫描探针 + 恢复配置
```

## 3. 数据流

### 3.1 烧录流程

```
用户点击"开始烧录"
  │
  ▼
FlashTab._on_flash_click()
  │
  ├── _validate() — 检查探针/芯片/固件是否已选
  ├── _stop_swo() — SWO 与烧录互斥，先停 SWO
  │
  ▼
FlashController.execute(task, on_progress)
  │
  ├── asyncio.to_thread(backend.connect)     → 连接目标
  │       │
  │       └── session_with_chosen_probe() + session.open()
  │           └── 返回 TargetInfo (Flash/RAM 布局)
  │
  ├── asyncio.to_thread(backend.program)     → 烧录
  │       │
  │       └── FileProgrammer(chip_erase="sector", smart_flash=True)
  │           └── progress → _make_progress_cb(emit)
  │               └── asyncio.run_coroutine_threadsafe → UI 更新
  │
  ├── asyncio.to_thread(backend.verify)      → 验证
  │       │
  │       └── 读回 Flash 数据 → 逐字节比对
  │
  ├── asyncio.to_thread(backend.reset)       → 复位
  │
  └── 返回 FlashResult → FlashPanel.set_complete()
```

### 3.2 SWO 数据流

```
SwoTab 开关打开
  │
  ├── backend.connect(target, probe, swv_config)
  │     └── session.options: enable_swv=True, swv_system_clock, swv_clock
  │
  ├── backend.swo_start_callback(sys_clock, swo_clock, callback)
  │     └── SWVReader(session).init() 
  │         ├── 配置 TPIU/ITM 寄存器
  │         ├── 启动后台线程
  │         └── 写入 _ConsoleWriter → callback(line)
  │
  └── callback → asyncio.run_coroutine_threadsafe → log_view.add_log("SWO", text)
```

### 3.3 日志流

```
任何层调用 add_log(level, message)
  │
  ▼
Logger 单例
  │
  ├── 存入内存缓冲 (最多 1000 条 → 保留最近 500)
  │
  └── 触发 callback (如果已注册)
       │
       └── asyncio.run_coroutine_threadsafe → log_view.add_log()
            │
            └── ft.Text(monospace, color=级别颜色) → ListView
```

## 4. 线程模型

```
┌─────────────── 主线程 (asyncio event loop) ───────────────┐
│  Flet UI 渲染 + 事件处理                                   │
│                                                            │
│  用户操作 → 启动后台任务:                                   │
│    page.run_task(coro)                                     │
│    asyncio.ensure_future(coro)                             │
│                                                            │
│  UI 更新来自后台线程:                                       │
│    asyncio.run_coroutine_threadsafe(update_fn, loop)       │
└────────────────────────────────────────────────────────────┘
         │                              ▲
         │ asyncio.to_thread()          │ run_coroutine_threadsafe
         ▼                              │
┌──────────────── 线程池 ────────────────────────────────────┐
│  pyOCD 阻塞 I/O:                                           │
│    backend.connect() / .program() / .erase_chip()          │
│    backend.swo_start_callback()                             │
│                                                            │
│  SWVReader 后台线程 (pyOCD 内部):                           │
│    TPIU/ITM 寄存器配置 → SWO 数据捕获 → 回调通知            │
└────────────────────────────────────────────────────────────┘
```

**关键规则：**
- 所有 pyOCD 阻塞调用必须经 `asyncio.to_thread()` 放入线程池
- 线程中更新 UI 必须经 `asyncio.run_coroutine_threadsafe()`
- 打包后严禁直接 `threading.Thread()` — 会冻结 UI

## 5. 后端接口

```
BackendABC (抽象基类)
├── list_probes() → list[ProbeInfo]
├── connect(target, probe_uid, frequency, pack_path, swv_config) → TargetInfo
├── disconnect()
├── erase_chip()
├── program(file_path, base_address, progress_callback)
├── verify(file_path, base_address) → bool
├── reset()
├── swo_start_callback(sys_clock, swo_clock, callback)
├── swo_stop()
├── get_target_info() → TargetInfo
├── is_connected → bool
└── swv_raw_port → int
```

当前唯一实现：`PyOCDBackend`，封装 pyOCD SDK。接口设计支持未来添加 `OpenOCDBackend` 等替代实现。

## 6. 数据结构

```python
ProbeInfo          # 探针信息 (name, unique_id, probe_type, description...)
TargetInfo         # 芯片信息 (name, part_number, flash_regions, ram_start, ram_size...)
FlashRegion        # Flash 区域 (name, start, length, sector_size, access)
FlashTask          # 烧录任务 (firmware_path, target_name, probe_uid, base_address, pack_path, erase_chip, swv_config)
FlashProgress      # 进度事件 (stage, percent, message)
FlashResult        # 烧录结果 (success, error_code, message, duration_seconds)
ErrorCode          # 错误码枚举 (0=成功, 1xxx=探针, 2xxx=连接, 3xxx=Flash, 4xxx=文件)
BackendError       # 后端统一异常 (携带 ErrorCode)
AppState           # 应用状态 (IDLE/SCANNING/CONNECTING/ERASING/PROGRAMMING/VERIFYING/ERROR)
LogEntry           # 日志条目 (timestamp, level, message)
```

## 7. 关键设计决策

| 决策 | 选型 | 理由 |
|------|------|------|
| UI 框架 | Flet (Flutter/Material 3) | 异步原生、跨平台桌面+Web、Apache-2.0 |
| 后端引擎 | pyOCD 0.44+ | 成熟、支持三大探针平台、CMSIS-Pack |
| 打包 | `flet build` | 一条命令多平台打包，无需配置 PyInstaller |
| 线程模型 | `asyncio.to_thread()` | Flet 官方推荐，避免打包后冻结 |
| SWO 实现 | SWVReader + 回调 | pyOCD 内置 ITM 解析，无乱码，无 TCP 层 |
| 芯片选择 | 厂家→芯片两级联动 | 减少单一下拉框 600+ 选项的浏览负担 |
| 配置持久化 | JSON 文件 | 轻量，用户目录 `%APPDATA%/mcu-cube-programmer/` |
